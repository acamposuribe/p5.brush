Async Multi-Instance Test

This page stress-tests the old report about multiple instance-mode sketches racing on shared global renderer state. Each worker sketch uses brush.instance(p); while it runs, tiny "stealer" sketches are created repeatedly to mimic asynchronous startup.

The first scenario matches the reported pattern more closely by binding the instance only once in the sketch function. The second scenario rebinds before each render pass as a control. Each worker checks both for thrown errors and for brush pixels landing on the correct canvas.
